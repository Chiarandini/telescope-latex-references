1. INTRO                                     *telescope-latex-labels-intro*

telescope-latex-labels.nvim is a Telescope extension for navigating LaTeX
labels across multi-file projects. It mirrors the architecture of
telescope-cached-headings but is specialised for recursive project scanning
and automatic label-prefix transformation.

On first use the project is scanned starting from the root `.tex` file,
following all `\include` and `\input` directives. The results are written to
a flat plaintext cache. Subsequent picker opens read the cache directly,
making them essentially instant even on large projects.

Two classes of label are supported out of the box:

  • Environment labels  `\begin{thm}{Snake Lemma}{snakeLem}`
    The environment name is mapped to a prefix via the `transformations`
    table (e.g. `thm` → `th:`), producing the id `th:snakeLem`.

  • Standard labels     `\label{id}`
    Context is inferred by looking back through the preceding lines for the
    nearest meaningful content (section title, question text, command name,
    or any custom LaTeX command). No whitelist needed.

3. SETUP                                      *telescope-latex-labels-setup*

Load and configure the extension inside your Neovim config: >

  require("telescope").setup({
    extensions = {
      latex_labels = {
        -- all keys are optional; defaults are shown
        cache_strategy    = "global",
        recursive         = true,
        auto_update       = false,
        enable_smart_jump = true,
        smart_jump_window = 200,

        -- Subfile toggle
        root_file          = "",
        subfile_toggle_key = "<C-g>",

        -- Copy label to clipboard
        copy_label_key = "<C-y>",
        copy_transform = nil,  -- table, function, or nil; see |latex-labels-copy|

        transformations = {
          thm      = "th:",
          prop     = "pr:",
          defn     = "df:",
          lem      = "lm:",
          cor      = "co:",
          example  = "ex:",
          exercise = "x:",
        },
        patterns = {
          { pattern = "\\begin{(%w+)}{(.-)}{(.-)}", type = "environment" },
          { pattern = "\\label{(.-)}", type = "standard" },
        },
      },
    },
  })

  require("telescope").load_extension("latex_labels")
<

Suggested keybindings: >

  vim.keymap.set("n", "<leader>fl",
    "<cmd>Telescope latex_labels<cr>",
    { desc = "Find LaTeX labels" })

  vim.keymap.set("n", "<leader>fL",
    "<cmd>LatexLabelsUpdate<cr>",
    { desc = "Update LaTeX labels cache" })
<

==============================================================================
4. USAGE                                      *telescope-latex-labels-usage*

Open the picker: >

  :Telescope latex_labels
<

  • The first time you open the picker, the project is scanned automatically
    starting from the root file. Subsequent opens read the cache directly.

  • Root file detection uses vimtex (`b:vimtex.tex`) when available, and
    falls back to the current buffer's file otherwise.

  • Each entry is displayed as:
      [label_id] :: context  (filename:line)

  • Selecting an entry triggers a *smart jump* (see |telescope-latex-labels-smart-jump|).
    The correct file is opened automatically when the label lives in a
    different file, and the view is centred with `zz`.

To force-rebuild the cache after adding or renaming labels: >

  :LatexLabelsUpdate
<

To delete every cache file and start fresh: >

  :LatexLabelsWipeAll
<

5. SMART JUMP                           *telescope-latex-labels-smart-jump*

When you select a label the plugin verifies whether the cached line number
still holds the expected label in the target file.

  Match    The label is exactly where the cache says it is. Jump silently.

  Shifted  The label is not at the cached line, but is found within
           ±`smart_jump_window` lines. Jump to the new position and patch
           the cache entry automatically.
           Message: "Label shifted. Cache auto-updated."

  Missing  The label cannot be found anywhere in the search window. Jump
           to the original cached line and warn the user.
           Message: "[Warning] Label not found at cached location. Please
           run :LatexLabelsUpdate."

Set `enable_smart_jump = false` to skip verification and jump directly to the
cached line number without any checks.

How labels are located ~

The search checks each candidate line for the substring `{label_id}`.
For prefix-transformed labels (e.g. id `th:snakeLem` produced from
`\begin{thm}{...}{snakeLem}`) the search also tries `{snakeLem}` — the
raw suffix after the last colon — so environment-inline labels are always
found even though the prefix does not appear in the source file.

If the file is already loaded as a Neovim buffer the search uses the buffer
API (zero disk I/O). For files not yet open, the search streams only the
relevant line window from disk so no extra buffer is opened.

6. LABEL PATTERNS                           *telescope-latex-labels-patterns*

                                      *latex-labels-environment-pattern*
Environment pattern ~

Matches labels declared inline on a single line with an explicit title: >

  \begin{thm}{Snake Lemma}{snakeLem}
<
The environment name (`thm`) is looked up in `transformations`. If a prefix
is found, the entry is stored with id `th:snakeLem` and context `Snake Lemma`.
If the environment is not in `transformations` the line is silently skipped
by this pattern (the standard pattern may still match a `\label{}` elsewhere
in the same environment).

                                        *latex-labels-standard-pattern*
Standard pattern ~

Matches any bare `\label{id}` regardless of surrounding context: >

  \Question[Why is the sky blue?]
    Some question text.
    \label{q:sky}
<
The plugin looks back through the 8 preceding lines for the first non-blank,
non-comment line and extracts a context string from it using the following
priority order:

  1. First brace group    `{content}`  →  content
  2. First bracket group  `[content]`  →  content
  3. Command name         `\cmd`       →  \cmd
  4. Raw line text (truncated to 80 characters)

This handles `\Question`, `\exercise`, `\section`, `\begin{...}`, and any
other custom LaTeX command without any additional configuration.

                                           *latex-labels-custom-patterns*
Custom patterns ~

Add entries to `transformations` for new environment names, and add entries
to `patterns` for environments with a different argument signature: >

  transformations = {
    thm = "th:",   -- built-in
    myEnv = "my:", -- custom
  },
  patterns = {
    -- built-in: \begin{env}{Title}{label}
    { pattern = "\\begin{(%w+)}{(.-)}{(.-)}", type = "environment" },
    -- custom: \begin{env}[label]{Title}
    { pattern = "\\begin{(%w+)}%[(.-)%]{(.-)}", type = "environment" },
    -- standard fallback (always keep last)
    { pattern = "\\label{(.-)}", type = "standard" },
  },
<

Pattern processing stops at the first `"environment"` match for a given line.
`"standard"` patterns do not stop processing.

7. SUBFILE TOGGLE                          *telescope-latex-labels-subfile*

When editing a LaTeX subfile (using the `subfiles` package), the plugin
detects a root file by checking three sources in order:

  1. vimtex — `b:vimtex.tex` when vimtex is active.
  2. `\documentclass[root.tex]{subfiles}` — parsed from the first 20 lines.
  3. `root_file` config key — an absolute path set manually as a fallback.

When the root file differs from the file you are editing the picker title
changes to reflect the current mode and the available toggle key:

  Full-project mode (default) ~
    Shows all labels from the entire project. The cursor is pre-positioned
    on the first label belonging to your current file. Title: >
      LaTeX Labels (full project) [<C-g>: this file]
<
  This-file mode ~
    Shows only labels defined in the current file. Title: >
      LaTeX Labels (this file) [<C-g>: full project]
<
Press `subfile_toggle_key` (default `<C-g>`) inside the picker to switch
between modes.

Manual root override: >

  latex_labels = {
    root_file          = "/home/user/thesis/main.tex",
    subfile_toggle_key = "<C-g>",
  }
<

8. COPY LABEL                                *telescope-latex-labels-copy*
                                                        *latex-labels-copy*

Press `copy_label_key` (default `<C-y>`) inside the picker to copy the
selected label's reference to the system clipboard (`+` register) without
opening the file.

copy_transform ~

An optional hook that transforms the label string before it is placed in the
clipboard. Accepts two forms:

Table — map label prefixes to Lua format strings (`%s` is the full label): >

  copy_transform = {
    ["df:"] = "\\cref{%s}",
    ["lm:"] = "\\cref{%s}",
    ["th:"] = "\\cref{%s}",
    ["co:"] = "\\cref{%s}",
    ["pr:"] = "\\cref{%s}",
    ["ex:"] = "example~\\ref{%s}",
    ["eq:"] = "equation~\\eqref{%s}",
  },
<
With this config pressing `<C-y>` on `df:grp` copies `\cref{df:grp}`, and
on `eq:euler` copies `equation~\eqref{eq:euler}`. Labels whose prefix is
not in the table are copied unchanged.

Function — for logic a flat table cannot express: >

  copy_transform = function(label)
    if label:match("^eq:") then
      return "equation~\\eqref{" .. label .. "}"
    end
    return "\\cref{" .. label .. "}"
  end,
<

When `copy_transform` is `nil` (the default) the raw label id is copied.

9. CONFIGURATION                           *telescope-latex-labels-config*

All options are passed in the `extensions.latex_labels` table during
`telescope.setup()`.

cache_strategy ~
    Type:    string
    Default: `"global"`

    Controls where cache files are stored.

    `"global"` (recommended)
      Stores all caches in Neovim's standard data directory:
        stdpath("data")/cached_labels/<hash>.labels
      Keeps your project directories clean.

    `"local"`
      Creates a hidden file next to the root .tex file:
        /path/to/dir/.main.tex.labels
      Useful if you want to inspect the cache. Remember to add `*.labels`
      to your `.gitignore`.

recursive ~
    Type:    boolean
    Default: `true`

    When `true`, the scanner follows `\include{file}` and `\input{file}`
    directives and recursively scans the referenced files. A cycle guard
    ensures each file is visited at most once.

    When `false`, only the root file is scanned.

auto_update ~
    Type:    boolean
    Default: `false`

    When `true`, the cache is regenerated every time you save any `.tex`
    file (`BufWritePost *.tex`).
    NOTE: For large multi-file projects this may cause a brief pause on save.

enable_smart_jump ~
    Type:    boolean
    Default: `true`

    See |telescope-latex-labels-smart-jump| for a full description.
    Set to `false` to always jump straight to the cached line number.

smart_jump_window ~
    Type:    integer
    Default: `200`

    The number of lines to search on each side of the cached position when
    a label is not found at its expected location.
    Only relevant when `enable_smart_jump = true`.

root_file ~
    Type:    string
    Default: `""`

    Absolute path to the root `.tex` file. Used as a manual fallback when
    vimtex is not active and the `\documentclass[...]{subfiles}` heuristic
    fails or is non-trivial. Has no effect if vimtex or the heuristic
    succeeds first.

subfile_toggle_key ~
    Type:    string
    Default: `"<C-g>"`

    Key (in both insert and normal mode) to toggle between full-project
    and this-file view inside the picker. Only active when the plugin has
    detected a root file that differs from the file you are editing.
    See |telescope-latex-labels-subfile|.

copy_label_key ~
    Type:    string
    Default: `"<C-y>"`

    Key (in both insert and normal mode) to copy the selected label
    reference to the system clipboard (`+` register) without navigating
    to the file.

copy_transform ~
    Type:    table | function | nil
    Default: `nil`

    Optional transformation applied to the label string before it is
    placed in the clipboard.

    When a table, keys are label-prefix strings and values are Lua format
    strings where `%s` is replaced by the full label id. The first
    matching prefix wins; unmatched labels are copied unchanged.

    When a function it receives the label string and must return the
    transformed string.

    When `nil` the raw label id is copied.

    See |telescope-latex-labels-copy| for full examples.

transformations ~
    Type:    table  (string → string)
    Default: see |telescope-latex-labels-setup|

    Maps LaTeX environment names to label prefixes. Used when the
    `"environment"` pattern matches a line. Environments not present in this
    table are silently skipped by the environment pattern; their labels may
    still be captured by the standard `\label{}` pattern.

patterns ~
    Type:    table  (list of { pattern, type })
    Default: see |telescope-latex-labels-setup|

    Ordered list of capture patterns applied to each line. Each entry has:

      `pattern`  A Lua pattern string.
      `type`     `"environment"` or `"standard"`.

    For `"environment"` patterns the pattern must capture three groups:
      (env_name, title, label)

    For `"standard"` patterns the pattern must capture one group:
      (label_id)

    Processing of `"environment"` matches stops (via `break`) on the first
    successful match for a line. `"standard"` patterns always run.

==============================================================================
10. COMMANDS                              *telescope-latex-labels-commands*

:Telescope latex_labels                         *:Telescope-latex_labels*

    Open the Telescope label picker for the current LaTeX project.
    Generates the cache on first use. The root file is determined via
    vimtex when available, otherwise the current buffer's file is used.

:LatexLabelsUpdate                                  *:LatexLabelsUpdate*

    Force-regenerate the cache for the current project by re-scanning from
    the root file. Use this after adding, renaming, or removing labels.
    Prints the number of labels found when complete.

:LatexLabelsWipeAll                                *:LatexLabelsWipeAll*

    Delete every cache file written by this plugin and print how many were
    removed. Useful when you want a guaranteed clean slate — for example
    after upgrading the plugin or after restructuring your project.

    The next time you open the picker the cache is regenerated automatically.

    Note: only supported with `cache_strategy = "global"`. With `"local"`
    the cache files sit next to your source files; delete them manually
    (they match the glob `*.labels`).

==============================================================================
12. CACHE FORMAT                          *telescope-latex-labels-cache*

Cache files are plain text — one label per line:

  line_number|label_id|context|absolute_filepath

Example: >

  50|th:snakeLem|Snake Lemma|/Users/me/thesis/chapter1.tex
  102|pr:bigProp|Main Proposition|/Users/me/thesis/chapter1.tex
  15|df:myDef|Definition of X|/Users/me/thesis/chapter2.tex
<

Lines beginning with `#` are treated as comments and ignored.

For the `"global"` strategy files live in:
  stdpath("data")/cached_labels/<sha256-of-root-path>.labels

For the `"local"` strategy the file is placed next to your root `.tex` file:
  /path/to/project/.main.tex.labels
